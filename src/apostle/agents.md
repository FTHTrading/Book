# The 20-Agent Commerce Mesh

## Agent Architecture

Every agent on the Apostle Chain is a fully autonomous economic actor. Each agent has:

- **Agent ID** — UUID registered on-chain (`agent:{uuid}`)
- **Ed25519 Keypair** — signs every transaction (SovereignKeyring)
- **ATP Balance** — native token balance on the Apostle ledger
- **Tier** — determines transaction amounts and cadence
- **Commerce Role** — what service the agent provides or consumes

## Agent Lifecycle

```
  ┌────────────────────────────────────────────────────────┐
  │                                                        │
  │  1. bootstrap.py generates Ed25519 keypair             │
  │  2. POST /v1/agents/register → on-chain identity       │
  │  3. POST /v1/airdrop → initial ATP funding             │
  │  4. agent_runner.py starts async loop:                 │
  │       ↳ fetch nonce → pick target → build tx →         │
  │         sign (Ed25519) → POST /v1/tx → websocket emit  │
  │  5. Repeat based on tier sleep interval                │
  │                                                        │
  └────────────────────────────────────────────────────────┘
```

## Tier Economics

| Tier | ATP per Tx | Sleep Range | Transactions/Hour |
|------|-----------|-------------|-------------------|
| **Operator** | 50 – 200 | 120 – 300s | 12 – 30 |
| **Control** | 5 – 25 | 30 – 90s | 40 – 120 |
| **Execution** | 1 – 10 | 10 – 40s | 90 – 360 |
| **Intelligence** | 2 – 15 | 20 – 60s | 60 – 180 |
| **Interface** | 0.5 – 5 | 15 – 45s | 80 – 240 |

## On-Chain Identity

Every agent's identity is entirely on-chain:

- **AgentId**: `AgentId(Uuid)` — bare UUID in serde, `agent:UUID` in display
- **Public Key**: Ed25519 public key (32 bytes, hex encoded)
- **Signature**: Ed25519 signature on TxEnvelope (64 bytes, hex encoded)
- **Nonce**: Monotonically increasing per-agent, prevents replay attacks
- **Chain ID**: 7332 — binds txs to Apostle Chain (prevents cross-chain replay)

> All 20 agent keypairs are stored in `keys/agents.json` — generated by `bootstrap.py` during initial deployment.
